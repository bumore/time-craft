# TimeCraft 托盘应用 — 实现计划

## 原则
不搞抽象层、不搞插件系统、不搞配置文件解析器。直接写能跑的代码。

## 架构

```
E:\bumoren\time-craft\
├── scripts\          （现有核心模块，不动）
│   ├── active_monitor.py
│   ├── browsing_monitor.py
│   └── input_hook.py
├── app\
│   ├── main.py       （入口：启动托盘 + 管理监控线程）
│   └── tray.py       （托盘图标、右键菜单、报告弹窗）
├── data\             （现有数据，不动）
└── requirements.txt
```

只新增 2 个文件，不重构现有脚本。

## 技术选型

- `pystray` — 托盘图标（~200KB，无重依赖）
- `Pillow` — 生成图标图片
- 现有脚本作为模块直接 import

## 右键菜单

```
📊 time-craft
├── ⏱ 活跃: 1h45m  ⌨️ 2797  🖱 73     ← 实时数据，不可点击
├── ─────────────────
├── 📋 查看报告                        ← 弹出 tkinter 窗口
├── ─────────────────
├── ✅ 监控中 [点击暂停]                ← 切换启停
├── ─────────────────
└── ❌ 退出
```

## 实现清单

| # | 任务 | 说明 |
|---|------|------|
| 1 | `requirements.txt` | pystray, Pillow |
| 2 | `app/tray.py` | 托盘图标 + 菜单 + 报告弹窗（tkinter） |
| 3 | `app/main.py` | 入口：启动监控线程 + 托盘，管理生命周期 |
| 4 | 测试运行 | 确认托盘、菜单、启停、报告都正常 |
| 5 | 打包 .exe | PyInstaller 单文件打包 |

## 不做

- 设置面板
- 开机自启配置
- 多语言
- 主题切换
- 数据库迁移

## 报告弹窗

用 tkinter（Python 自带），零额外依赖，够用。
